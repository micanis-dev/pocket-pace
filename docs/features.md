# 機能一覧

## 1. 口座・カード管理

### 1.1 口座の登録・管理

銀行口座、財布、タンス預金など、現金や残高を持つ場所を登録できる。

登録できる項目の例は以下のとおり。

- 口座名
- 種類
  - 銀行口座
  - 財布
  - タンス預金
  - その他
- 現在残高

### 1.2 カードの登録・管理

クレジットカードを登録できる。

登録できる項目の例は以下のとおり。

- カード名
- 締め日
- 引き落とし日
- 引き落とし口座

### 1.3 口座間の資金移動

口座間のお金の移動を記録できる。

例は以下のとおり。

- 銀行口座から財布に現金を下ろした
- 財布から貯金用の封筒に移した
- メイン口座から貯金用口座に移した

これは支出ではなく、資産の置き場所が変わっただけとして扱う。

### 1.4 カード利用額と支出記録の突き合わせ

クレジットカード払いを日々の支出として記録した場合、

カード請求額をそのまま支出として扱うと二重計上になる。

そのため、以下の管理が必要である。

- 日々登録したカード支出の合計
- 実際のカード請求額
- 差額の確認
- 差額がある場合の調整

カード請求額は支出そのものではなく、過去に記録した支出の支払いとして扱う。

## 2. 収入管理

### 2.1 給与の登録

給与の金額と給料日を登録できる。

登録できる項目の例は以下のとおり。

- 給与名
- 金額
- 給料日
- 入金先口座

### 2.2 給与の入力方式

給与の入力方式は、以下から選択できる。

- 毎月自動で同じ金額を追加する
- 給料日に通知を出し、手動で追加する

毎月の給与額が安定している場合は自動入力、

変動がある場合は手動入力を選べるようにする。

### 2.3 入力忘れ防止アラート

給料日を過ぎても収入が入力されていない場合、通知を出す。

### 2.4 臨時収入の記録

給与以外の収入を記録できる。

例は以下のとおり。

- アルバイト代
- 副業収入
- お小遣い
- 売却益
- 返金

### 2.5 繰り越し

前の周期で余ったお金を、次の周期でどう扱うか選べる。

選択肢は以下のとおり。

- 次の周期の使用可能額に追加する
- 貯金に回す
- 繰り越さず、残額として保持する

## 3. 支出管理

### 3.1 固定費・サブスクの自動入力

毎月決まって発生する支出を自動で登録できる。

例は以下のとおり。

- 家賃
- スマホ代
- サブスク
- 保険料
- 通信費

支払い元も管理できるようにする。

- 銀行口座から引き落とし
- クレジットカード払い
- 現金払い

### 3.2 金額未確定の固定費予定

光熱費や水道代など、金額は未確定だが必ず発生する支出を予定として登録できる。

この場合、以下のように扱う。

- 予定支出として仮に表示する
- 金額が確定したら実支出に変更する
- 使用可能額には、仮の金額を反映するか選べる

### 3.3 変動費の記録

日々の支出を記録できる。

登録できる項目の例は以下のとおり。

- 金額
- 日付
- カテゴリ
- 支払い方法
- 支払い元
- メモ

### 3.4 クレジットカード利用時の即時反映

クレジットカード払いでも、今期の使用可能額からすぐに差し引く。

これにより、カード払いによる使いすぎを防ぐ。

### 3.5 クレジットカード請求額の調整

カード請求額と、日々記録したカード支出の合計が一致しない場合、差額を調整できる。

差額の扱いは以下のようにする。

- 不明な支出として追加する
- 手数料・年会費として追加する
- 既存の支出記録を修正する
- 差額を無視する

## 4. 貯金管理

### 4.1 毎月の貯金ルール設定

毎月どのくらい貯金するかを設定できる。

設定方式は以下のとおり。

- 固定金額
- 収入の一定割合
- 毎回手動で決める変動制

### 4.2 目的別貯金

目的別に貯金を管理できる。

例は以下のとおり。

- 旅行用
- 車用
- 引っ越し用
- 緊急用
- 欲しいもの用

### 4.3 貯金の振り分け

設定したルールに従って、貯金額を各目的に各収入から振り分けられる。
あくまでもアプリ上での目安となる。

## 5. 予算管理

### 5.1 予算リセット周期の設定

Pocket-Pace では、基本的に1か月単位で使用可能額をリセットする。

この1か月の区切り方は、ユーザーが選択できる。

設定できる例は以下のとおり。

- 月初から月末まで
- 給与日から次回給与日の前日まで
- 任意の開始日から最長1か月後の終了日まで

この周期は、使用限度額を計算するための基準期間である。

### 5.2 表示・分割粒度の設定

1か月の使用可能額を、どの粒度で分割して表示するかを設定できる。

設定できる例は以下のとおり。

- 1日単位
- 1週間単位
- 2週間単位
- 1か月単位

例えば、1か月の使用可能額が90,000円で、表示粒度が1日単位の場合、
1日あたり約3,000円使えるものとして表示する。

表示粒度が1週間単位の場合は、
1週間あたりに使える金額を表示する。

表示粒度が2週間単位の場合は、
2週間あたりに使える金額を表示する。

表示粒度が1か月単位の場合は、
今月全体であといくら使えるかを中心に表示する。

この設定は、予算をリセットする周期とは別の概念である。

### 5.3 使用限度額の自動計算

周期ごとの使用限度額を自動計算する。

基本式は以下のとおり。

```text
使用限度額 = 総収入 + 繰り越し - 固定費 - 予算反映対象の予定支出 - 貯金額
```

カード払いも、支払日ではなく利用日を基準に使用限度額へ反映する。

固定費、および使用限度額へ反映済みの予定支出から変換した実支出は、控除済みのため残額から再度差し引かない。予定支出の金額が確定した場合は、予定額を確定額へ置き換えて使用限度額を再計算する。

### 5.4 残額とペース表示

現在の周期であといくら使えるかを表示する。

表示する内容の例は以下のとおり。

- 今期の使用限度額
- 現在の残額
- 設定した表示粒度ごとの使用可能額
- 現在の消費ペース
- 予定より使いすぎているかどうか

## 6. ユーザー設定

### 6.1 見た目のカスタマイズ

アプリの見た目を変更できる。

例は以下のとおり。

- ライトモード
- ダークモード
- システム設定に合わせる
- テーマカラー

### 6.2 支出カテゴリのカスタマイズ

支出カテゴリを自由に追加・編集・削除できる。

例は以下のとおり。

- 食費
- 交通費
- 娯楽
- 日用品
- 医療費
- サブスク
- 交際費

### 6.3 通知設定

通知のオン・オフや通知タイミングを設定できる。

通知の例は以下のとおり。

- 給料日の入力通知
- 固定費の支払い予定通知
- 使用ペース超過通知
- 周期終了前の残額通知
- 入力忘れ通知
